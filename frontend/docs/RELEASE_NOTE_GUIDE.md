# 更新公告规范

每次修 bug 或新增功能后，开发者**必须**在 `src/data/releaseNotes.js` 中新增一条公告，让用户知道发生了什么变化。

## 版本号规则

每次发布按以下规则递增：

### 小修复 / 小优化（patch）
最后一个数字 +1。适用于单页面小 bug、文案修正、样式微调。

```
v1.1.1 → v1.1.2
```

### 中等范围修复 / 优化（minor）
中间数字 +1，最后一位回到 1。适用于多个页面交互打磨、做题流程优化、导航体验整体修复。

```
v1.1.1 → v1.2.1
```

### 大版本（major）
直接使用 v2。适用于整体架构升级、重大视觉改版、核心流程重做。

```
v1.x.x → v2
```

## 数据结构

```js
{
  id: "v1.1.1-short-description",       // 唯一 ID（版本-简短英文）
  version: "v1.1.1",                    // 当前版本号
  date: "2026-06-20",                   // 发布日期
  type: "优化",                          // 见下方 type 说明
  title: "优化导航与学习概览",            // 一句话标题
  items: [                              // 最多 3 条简短说明
    "底部导航新增 AI 对话入口",
    "练习入口移至首页",
    "我的页新增学习概览入口",
  ],
}
```

## type 可选值

| type | 使用场景 |
|------|----------|
| `修复` | 修复用户可见的 bug |
| `新增` | 增加新功能 |
| `优化` | 现有功能改进、UI 调整 |
| `重要` | 重大变更、需用户注意 |

## 文案规范

- 每条公告言简意赅，用户能看懂
- **不写内部技术细节**（文件名、函数名、框架名、接口名）
- 用用户视角写：用户看到了什么、体验了什么
- 每条公告的 items 控制在 **3 条以内**
- **公告内容必须和本次实际改动一致，不要把未改动页面写进公告**

### 示例

```
修复：修复 AI 导入后题库名不同步的问题
     × "修复了 aiImportTask 中 courseName 未正确设置的问题"

新增：新增学习概览详情页
     × "新增 StudyOverview.vue 页面和 useStudyOverview composable"

优化：优化底部导航，减少重复入口
     × "移除了 list 和 practice 路由的导航标签"
```

## 规则

1. 每次修 bug → 新增一条 `修复` 公告
2. 每次加功能 → 新增一条 `新增` 公告
3. 每次 UI/UX 改版 → 新增一条 `优化` 公告
4. 变更影响旧数据或需要用户手动操作 → 新增一条 `重要` 公告
5. **禁止只改代码不写公告**
6. **禁止空 commit message + 无公告**
